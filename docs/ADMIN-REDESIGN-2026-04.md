## 2. 설계 철학

### 핵심 원칙: **"3초 안에 상황 파악, 1클릭으로 해결"**

1. **한 화면 = 한 목적**: 각 화면이 정확히 하나의 질문에 답한다
2. **점진적 공개**: 요약 → 상세 → 조치 (3단계)
3. **색상 = 상태**: 초록(정상) / 노랑(주의) / 빨강(긴급) — 글을 읽기 전에 색으로 파악
4. **최신화 버튼 = 최상위**: 페이지 어디서든 접근 가능
5. **모바일 퍼스트**: 세로 스크롤 1열 기본, 태블릿/데스크탑에서 2~3열

## 3. 새 구조: 5개 탭

### 기존 14개 → 5개로 통합

```
┌─────────────────────────────────────────────────────────┐
│  🏠 현황    📊 성장    🔧 운영    📋 콘텐츠    ⚡ 실행   │
└─────────────────────────────────────────────────────────┘
```

| 새 탭 | 흡수하는 기존 탭 | 핵심 질문 |
|-------|----------------|-----------|
| **🏠 현황** | 대시보드 + 방문자 + SEO | "지금 카더라는 어떤 상태야?" |
| **📊 성장** | (신규) 전환 추적 + 유저 분석 | "전환율은 올라가고 있어?" |
| **🔧 운영** | 시스템 + 신고/결제 + 공지 + 팝업 | "고장난 데 없어?" |
| **📋 콘텐츠** | 콘텐츠 + 블로그 + 부동산 + 위성 + 상점 | "데이터가 제대로 돌아가고 있어?" |
| **⚡ 실행** | GOD MODE + 전수조사 | "한 버튼으로 전체 최신화" |

## 9. `🚀 전체 최신화` 버튼 상세 스펙

### 동작 흐름

```
1. 버튼 클릭
   ↓
2. 확인 모달: "83개 크론을 5단계로 실행합니다. 계속하시겠습니까?"
   ↓
3. POST /api/admin/god-mode { mode: 'full' }
   ↓
4. 실행 탭으로 자동 전환
   ↓
5. 실시간 진행 표시 (3초마다 cron_logs 폴링)
   ↓
6. 완료: "✅ 79/83 성공 · ❌ 4 실패 · 소요 4분 32초"
   ↓
7. [실패만 재실행] 버튼 표시
```

### 상태 표시 (플로팅 바)

```
실행 전:  ⚡ 전체 최신화          ← 기본
실행 중:  ⚡ 실행 중... 45/83     ← 파란색 맥박 애니메이션
완료:     ⚡ ✅ 완료 (79/83)      ← 3초 후 기본으로 복귀
에러:     ⚡ ⚠️ 4건 실패          ← 노란색, 클릭하면 실패 목록
```

### 스마트 실행 (향후)

```
"스마트 최신화" 모드:
- 마지막 성공 시각 기준으로 실행 필요한 크론만 선별
- 예: stock-crawl이 5분 전에 성공했으면 스킵
- 예: blog-rewrite가 2시간 전에 실패했으면 우선 실행
- 크론별 cooldown 설정: stock-crawl 10분, blog-rewrite 30분
```
