## 2. 근본 원인 분석

### 2.1 CTA를 볼 수 없는 구조적 문제

```
근본 원인: GuestNudge 최소 트리거 = 2PV
현실:      방문자 90%가 1PV에서 이탈
결과:      99.2% 방문자가 CTA를 한 번도 안 봄
```

GuestNudge가 사실상 유일한 전역 CTA인데, 2PV 이상 조건이 방문자 대부분을 걸러냄.
블로그/부동산 상세 페이지의 CTA들(SmartSectionGate, TwoStepCTA)은 스크롤해야 보이므로 바운스 사용자에겐 무의미.

### 2.2 주식 페이지 CTA 부재

주식 페이지(298 PV)에 TwoStepCTA, SmartSectionGate 등 전환 컴포넌트가 전혀 없음.
종목 상세 페이지에서 "이 종목 알림 받기" CTA가 없어서 전환 기회 0.

### 2.3 메시지 일반성

```
현재: "주식 알림 · 청약 알림 · 실시간 토론"  (일반적)
이상: "삼성전자 급등/급락 시 알림 받기"      (맥락 맞춤)
```

사용자가 삼성전자 페이지를 보고 있는데 "주식 알림"이라고 하면 와닿지 않음.
"삼성전자가 ±5% 변동하면 바로 알려드립니다"가 훨씬 전환율 높음.

### 2.4 추적 사각지대

BlogTopBanner, BlogMidCTA, BlogFloatingCTA가 trackConversion을 호출하지 않거나,
노출 이벤트(cta_view)를 기록하지 않아 실제 노출 수를 측정할 수 없음.

## 3. 개선 계획

### Phase 1: 즉시 효과 — CTA 노출률 극대화 (1일)

#### 3.1.1 GuestNudge 트리거 조건 완화

```
변경 전: 2PV+ → 토스트, 5PV+ → 모달
변경 후: 1PV (15초 체류) → 토스트, 2PV+ → 배너, 4PV+ → 모달
```

- **1PV 15초 체류 후 토스트 표시**: 대부분의 방문자에게 최소 1회 CTA 노출 보장
- 세션당 토스트 1회 유지 (스팸 방지)
- SEO 상세 페이지(/blog/*, /apt/*, /stock/*)에서도 토스트 허용 (현재 일부 차단)

예상 효과: 노출률 0.78% → 15~25%

#### 3.1.2 주식 페이지에 TwoStepCTA 추가

```tsx
// stock/[symbol]/page.tsx에 추가
{!userId && <TwoStepCTA category="stock" contextName={s.name} />}
```

- 종목명이 CTA 질문에 자동 반영: "삼성전자 시세가 급변하면 알림 받으시겠어요?"
- AI 분석 섹션 바로 아래 배치 (관심이 가장 높은 지점)

#### 3.1.3 ExitIntentPopup 체류 조건 완화

```
변경 전: 10초 후 활성화
변경 후: 5초 후 활성화
```

- 평균 체류시간이 짧으므로 5초로 줄여야 트리거됨

### Phase 2: 메시지 맞춤화 — 전환율 향상 (1~2일)

#### 3.2.1 맥락 기반 동적 CTA 메시지

```
페이지 유형별 메시지 동적 생성:

블로그 (/blog/stock-samsung-analysis):
  → "삼성전자 AI 분석이 업데이트되면 알림 받기"

종목 (/stock/005930):
  → "삼성전자(005930) 가격 변동 알림 받기"
  → "현재가 193,100원 — ±5% 변동 시 알림"

현장 (/apt/힐스테이트-부산):
  → "힐스테이트 부산 청약 마감 D-7 알림"
  → "분양가 2.5억~5.8억 — 가격 변동 알림"
```

구현: GuestNudge와 TwoStepCTA에 현재 페이지의 종목명/현장명을 props로 전달.
stock/[symbol] → `contextName={s.name}`, apt/[id] → `contextName={name}`.

#### 3.2.2 블로그 SmartSectionGate 헤딩 패턴 확장

```
변경 전: ['향후 전망', '투자 의견', '목표가', 'AI 분석'] 등 12개
변경 후: + ['핵심 요약', '결론', '투자 전략', '매매 전략', '체크리스트',
           '비교 분석', '수익률 시뮬레이션', '실전 적용', '입주 전략'] 15개 추가
```

현재 AI가 생성하는 블로그의 헤딩 패턴을 분석해서, 실제로 사용되는 섹션 제목을 추가.

#### 3.2.3 피드 페이지 CTA 추가

피드(PV/UV 1.3으로 가장 높은 참여도)에 맞춤 CTA:
- 피드 3번째 게시글 사이에 인라인 CTA 삽입
- "이 토론에 참여하려면 카카오 가입 (3초)"

### Phase 3: 추적 정비 — 측정 가능한 퍼널 (1일)

#### 3.3.1 모든 CTA에 trackConversion 연동

누락된 컴포넌트:
- BlogTopBanner: cta_view 이벤트 추가
- BlogMidCTA: cta_view 이벤트 추가
- BlogFloatingCTA: cta_view 이벤트 추가

#### 3.3.2 가입 완료 추적

현재 가입 이벤트를 conversion_events에 기록하지 않음.
로그인 콜백에서 신규 가입 시 `cta_complete` 이벤트 기록:

```ts
// auth callback에서
if (isNewUser) {
  trackConversion('cta_complete', 'signup', { 
    category: redirectPath?.startsWith('/stock') ? 'stock' : 
              redirectPath?.startsWith('/apt') ? 'apt' : 'general' 
  });
}
```

#### 3.3.3 어드민 대시보드 "전환" 라벨 수정

```
변경 전: "전환 3" (CTA 이벤트 수 — 오해 유발)
변경 후: "CTA 3" 또는 세분화:
         "노출 93 | 클릭 1 | 가입 14"
```

### Phase 4: A/B 테스트 인프라 (향후)

#### 3.4.1 CTA 변형 테스트

visitor_id 기반으로 A/B 그룹 분리:
- A: 현재 "카카오로 3초 가입"
- B: "무료로 알림 받기"
- C: "이 정보 저장하기"

conversion_events에 variant 컬럼 추가하여 그룹별 전환율 비교.

## 4. 예상 효과

### Before (현재)

```
노출률:   0.78% (66/8,490)
클릭률:   0% (실질)
전환률:   0.16% (14/8,490) — CTA 무관 자발적 가입
```

### After (Phase 1+2 적용 후 예상)

```

클릭률:   2~5% (맥락 맞춤 메시지)
전환률:   0.5~1.0% (현재 대비 3~6배)

일일 기준:
  방문자 ~1,200 UV/일
  CTA 노출: 180~300명
  CTA 클릭: 4~15명
  가입: 6~12명 (현재 2명 → 3~6배)
```
