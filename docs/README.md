# 카더라 작업보고서 (docs/)

## 이 폴더는 뭔가요?

카더라 웹앱 개발 세션의 작업 기록을 관리하는 폴더입니다.
새 세션을 시작할 때 **STATUS.md 하나만 읽으면** 현재 프로젝트 상태를 바로 파악할 수 있습니다.

## 폴더 구조

```
docs/
├── STATUS.md              ← 항상 최신 상태 (새 세션 시작 시 이것만 읽기)
├── README.md              ← 이 파일
└── sessions/
    ├── session-16.md      ← 세션별 상세 작업 기록
    ├── session-17.md
    └── session-XX.md      ← 계속 추가
```

## 사용법

### 새 세션 시작할 때
```
"docs/STATUS.md 읽고 작업 이어가자"
```
이 한 마디면 됩니다.

### 세션 끝날 때
1. `docs/sessions/session-XX.md` — 이번 세션 작업 내용 기록
2. `docs/STATUS.md` — 최신 상태로 업데이트 (미해결 사항, DB 현황, 배포 상태 등)
3. 커밋 & 푸시

### 두 컴퓨터 동시 작업 시
- 작업 중에는 각자 세션 파일(`session-XX.md`)만 작성
- `STATUS.md`는 **마지막에 끝나는 쪽**이 업데이트
- 작업 시작 전 `git pull`로 최신 상태 확인

## 세션 파일 작성 규칙

파일명: `session-{번호}.md`

기본 구조:
```markdown
# 카더라 세션 {번호} 작업 요약
**날짜:** YYYY-MM-DD
**최신 커밋:** `{SHA}` → 배포 `{dpl_ID}` (상태)

## 배포 이력
## 주요 변경사항
## DB 변경사항
## 미해결 / 다음 세션 작업
```
