## 2. 경쟁 환경 분석

### 2.1 부동산 현장명 검색 (블루오션)

"OO아파트 청약/분양/분양가" 검색 시:
- **1위:** 호갱노노 or 시공사 공식 사이트 (50%)
- **2위:** 개인 블로거 분석글 1~2개 (40%)  
- **3위:** 뉴스 기사 (10%)
- **경쟁자 총 3~5개** → 제대로 된 콘텐츠 1편이면 진입 가능

카더라의 경쟁 우위:
- 모집공고 파싱 데이터 (개인 블로거는 수동 타이핑)
- 분양가 vs 실거래가 비교 (자동 차트)
- 입주비용/청약가점 계산기 (경쟁자 없음)
- 같은 시공사 전국 비교 (경쟁자 없음)

### 2.2 주식 롱테일 검색 (경쟁 보통)

"OO종목 배당금/비교/분석" 검색 시:
- **1위:** 전문 금융 블로거 (moneydaylife, mcheam 등)
- **2위:** investing.com, stockevents 등 해외 플랫폼
- **3위:** 뉴스 기사
- **경쟁자 5~10개** → 더 풍부한 도구+데이터로 차별화 필요

### 2.3 핵심 인사이트

> **"데이터 대시보드"로는 절대 1위 못함. "분석 + 가이드 + 도구"가 있는 페이지만 1위.**
> 
> 카더라는 "도구"가 압도적으로 강함 (계산기 142종).
> "분석"과 "가이드"를 AI로 자동 생성하면 삼각편대 완성.

```

     ↑                    ↓                        ↓                       ↓
 [키워드 타겟]      [내부 링크]              [체류시간 ↑]           [D.I.A 점수 ↑]
```

3. **도구 연결** — 계산기/시뮬레이터를 SEO 무기로 활용

## 5. Phase 1: 부동산 현장 페이지 강화 (P0 — 1주차)

### 5.1 AI 분석 텍스트 자동 생성

**크론명:** `apt-analysis-gen`  
**스케줄:** 매일 05시  
**처리량:** 50건/일 (Batch API)  
**대상:** analysis_text가 NULL인 apt_sites (5,738건 → 약 115일 전체 커버)  
**우선순위:** page_views DESC, interest_count DESC (인기 현장 우선)

**AI 프롬프트 구조:**

입력 데이터 (apt_sites + apt_subscriptions + apt_transactions JOIN):
- 현장명, 위치, 시공사, 세대수, 분양가, 입주일
- 경쟁률, nearby_facilities, nearby_station, school_district
- price_comparison (주변 시세), transit_score
- payment_schedule, option_costs, extension_cost

출력 구조 (2,000자+, 마크다운):
```
## {현장명} 입지 분석
{교통 접근성 분석: 인근역 + 거리 + 노선 / 도로 + 고속도로 / 개발 호재}
{학군 분석: 배정 학교 + 거리 + 학업 성취도 (있다면)}
{생활 편의: nearby_facilities 기반 분석}

## 분양가 분석
{분양가 범위 해석: price_min~price_max}
{주변 시세 대비: price_comparison 데이터 기반}
{평당가 비교: 같은 시공사/같은 지역 대비}
{시세차익 전망: 주변 실거래가 - 분양가 차이 분석}

## 청약 전략
{경쟁률 분석: 과거 경쟁률 + 유사 현장 비교}
{유리한 타입: 면적별 경쟁률 차이 분석}
{가점 컷 예측: 지역 + 세대수 기반}

## 입주 준비
{필요 자금: 계약금 + 중도금 + 잔금 + 옵션 + 취득세}
{입주비용 시뮬레이션 안내: /calc 링크}

## 자주 묻는 질문
{FAQ 5개: 실제 검색되는 질문 기반}
```

**핵심:** 모든 분석이 **카더라 DB 실제 데이터 기반**이라 개인 블로거보다 정확하고 구체적.

### 5.2 apt/[id] 페이지 SSR 적용

```tsx
// analysis_text가 있으면 서버 렌더링으로 추가
{site?.analysis_text && (
  <section className="apt-analysis">
    <div dangerouslySetInnerHTML={{ __html: renderMarkdown(site.analysis_text) }} />
  </section>
)}
```

렌더링 위치: 모집공고 요약 섹션 바로 아래 (메인 콘텐츠 영역)

### 5.3 내부 링크 자동 삽입

analysis_text 내에 자동 삽입되는 내부 링크:
- `[청약 가점 진단하기](/apt/diagnose)` → 계산기 연결
- `[입주비용 계산하기](/calc/move-in-cost)` → 계산기 연결
- `[{시공사} 다른 현장 보기](/apt/sites/{builder-slug})` → 시공사 허브
- `[{지역} 실거래가 보기](/apt/complex?region={region})` → 단지백과 연결
- `[{현장명} 관련 블로그](/blog?q={name})` → 관련 블로그 연결


네이버 C-Rank에서 "이 사이트는 부동산 전문"이라고 인식하게 만듦.

### 8.2 naver:written_time 최적화

| 페이지 유형 | 현재 | 변경 |
|-----------|------|------|
| apt/[id] | site.created_at (DB 입수일) | 모집공고일 or 청약접수일 |
| stock/[symbol] | stock.updated_at (시세 갱신) | 분석 텍스트 생성일 |
| blog/[slug] | rewritten_at or published_at | 변경 없음 (이미 최적) |

### 8.3 네이버 웹마스터 수동 제출

우선순위:
1. 인기 현장 TOP 100 → URL 수동 제출
2. 계산기 메인 페이지 142개 → URL 수동 제출
3. 인기 종목 TOP 50 → URL 수동 제출
4. 이후 IndexNow 크론이 자동 처리

### 8.4 네이버 스마트 블록 최적화

네이버 검색 결과의 "VIEW" 영역은 스마트 블록으로 구성됨.
블로그 콘텐츠가 스마트 블록에 노출되려면:

- 제목에 **구체적 키워드** 포함 (X: "아파트 분석", O: "힐스테이트 동탄 분양가 분석")
- 본문 **2,500자 이상**
- **이미지 3장 이상** (OG 이미지 + 차트 캡처 + 데이터 표)
- **최신성** (작성 후 7일 이내 노출 확률 높음)

## 11. 자동화 파이프라인 전체 설계

### 11.1 신규 현장 등록 시 자동 파이프라인

```
[apt-crawl-pricing 크론] 신규 현장 감지
  ↓
[apt-parse-announcement 크론] 모집공고 파싱
  ↓
[sync-apt-sites 크론] apt_sites 레코드 생성
  ↓ (다음 날)
[apt-analysis-gen 크론] AI 분석 텍스트 생성 (2,000자+)
  ↓ (같은 날)
[blog-apt-cluster 크론] 클러스터 블로그 5편 생성
  ↓ (같은 날)
[blog-internal-links 크론] 상호 내부 링크 처리
  ↓ (같은 날)
[indexnow-mass 크론] 네이버/구글에 URL 제출
```

총 소요: 신규 현장 등록 후 **48시간 이내** 검색엔진에 완전한 콘텐츠 제출.

### 11.2 기존 현장 갱신 파이프라인

```
[주 1회] apt-analysis-gen: 실거래가 변동/경쟁률 업데이트 반영
  ↓
[자동] naver:updated_time 갱신 → 네이버 최신성 점수 ↑
  ↓
[자동] IndexNow 재제출
```

### 11.3 신규 콘텐츠 적용 방안

모든 AI 생성 콘텐츠에 적용되는 규칙:

1. **프롬프트 다양성:** blog-prompt-diversity.ts v2 (32종 조합) 적용
2. **FAQ 품질:** 실제 네이버 자동완성 키워드 기반 질문 생성
3. **데이터 구체성:** 모든 숫자는 DB 실제값 사용 (AI 환각 방지)
4. **내부 링크:** 최소 3개 (현장 페이지 + 계산기 + 관련 블로그)
5. **이미지:** OG 이미지 자동 생성 (이미 safeBlogInsert에서 처리)
6. **naver:written_time:** 작성 시점 정확히 기록

## 12. 리스크 분석

### 12.1 기술적 리스크

| 리스크 | 확률 | 영향 | 대응 |
|--------|------|------|------|
| AI 분석 텍스트 환각 (잘못된 숫자) | 중 | 높음 | 모든 숫자는 DB에서 직접 삽입, AI는 해석만 |
| analysis_text가 너무 비슷 (템플릿 감지) | 중 | 높음 | prompt-diversity 32종 + 현장별 데이터 차이로 자연 분산 |
| Anthropic 크레딧 소진 | 낮음 | 중간 | Batch API 50% 할인, 월 $20 내외 |
| Supabase DB 용량 초과 | 중 | 높음 | analysis_text 평균 4KB × 7,500건 = 30MB (무시 가능) |
| 빌드 시간 증가 | 낮음 | 낮음 | ISR이라 빌드 시 전체 렌더 안 함 |

### 12.2 SEO 리스크

| 리스크 | 확률 | 영향 | 대응 |
|--------|------|------|------|
| 네이버가 AI 생성 콘텐츠 감지 | 중 | 높음 | 데이터 기반 분석이라 "경험적" 가치 있음. 순수 AI 텍스트가 아닌 "데이터 해석" |
| 구글 Helpful Content 강등 | 낮음 | 높음 | 계산기/도구 연결로 "unique value" 제공 — 블로거 복붙과 차별화 |
| 과도한 콘텐츠 폭발 (스팸 감지) | 중 | 높음 | 클러스터 블로그 1일 10편 제한, safeBlogInsert 품질 관리 |
| 경쟁자 따라하기 | 낮음 | 낮음 | 데이터+계산기+AI 삼각편대는 진입장벽 높음 |

### 12.3 비즈니스 리스크

| 리스크 | 확률 | 영향 | 대응 |
|--------|------|------|------|
| 트래픽 유입되지만 전환 없음 | 중 | 중간 | 회원가입 CTA + 알림 설정 + 계산기 사용 유도 |
| 부정확한 분양가/시세 정보로 민원 | 낮음 | 높음 | 면책 문구 + "공식 정보는 청약홈 확인" 안내 |
| 저작권 이슈 (모집공고 내용) | 낮음 | 중간 | 공공데이터 기반, 요약/분석 형태로 fair use |
