## 3. 데이터 파이프라인 설계 — 어떻게 가져올 것인가

### 3.1 주식 데이터 파이프라인

```
Layer 1: 원천 데이터 수집 (매일)
┌─────────────────────────────────────────────────────────┐
│ stock-crawl (KRX API)                                   │
│   → 가격, 변동률, 거래량, 시총 (한국)                      │
│   → 매일 09시, 16시 (장 시작/마감)                        │
│                                                          │
│ stock-fundamentals-kr (신규) ← 네이버 증권 스크래핑         │
│   → PER, PBR, 배당수익률, 52주고저 (한국)                  │
│   → 매일 17시 (장 마감 후)                                │
│   → 소스: m.stock.naver.com/api/stock/{symbol}/basic      │
│                                                          │
│ stock-fundamentals-us (신규) ← Yahoo Finance API          │
│   → PER, PBR, EPS, ROE, 배당, 52주고저, 시총 (미국)        │
│   → 매일 07시 (미국 장 마감 후 KST 기준)                   │
│   → 소스: query1.finance.yahoo.com/v8/finance/quoteSummary │
│   → 모듈: summaryDetail, defaultKeyStatistics             │
│                                                          │
│ stock-sector-fill (신규) ← 네이버 + Yahoo                  │
│   → 섹터/산업 분류 (빈 종목 대상)                           │
│   → 주 1회                                                │
└─────────────────────────────────────────────────────────┘

Layer 2: 데이터 검증 (수집 직후)
┌─────────────────────────────────────────────────────────┐
│ stock-data-validate (신규)                               │
│   ① 가격 검증: 전일 대비 ±50% 이상 변동 → 이상치 플래그     │
│   ② 시총 검증: 가격 × 발행주식수 vs 저장된 시총 (±30%)     │
│   ③ PER 범위: 0 < PER < 500 (그 외 이상치)               │
│   ④ 배당률 범위: 0% ~ 30% (그 외 이상치)                  │
│   ⑤ NULL 체크: 핵심 지표 NULL이면 재수집 대상 표시         │
│   → 결과: data_quality_alerts 테이블에 기록               │
│   → 매일 18시                                            │
└─────────────────────────────────────────────────────────┘

Layer 3: 파생 데이터 생성 (검증 후)
┌─────────────────────────────────────────────────────────┐
│ stock-analysis-gen (기존)                                │
│   → 정확한 PER/PBR/시총 데이터 기반 AI 분석 생성           │
│   → 매시간 15건                                          │
│                                                          │
│ analysis-refresh (기존)                                   │
│   → 7일 지난 분석 + 데이터 변동 큰 종목 우선 재생성         │
│   → 매주 일요일                                          │
└─────────────────────────────────────────────────────────┘
```

```
┌─────────────────────────────────────────────────────────┐

│   ② 세대수 이상치: 0 또는 50,000 초과 → 플래그            │
│   ③ 좌표 이상치: 대한민국 범위(33~39°N, 124~132°E) 벗어남  │
│   ④ 시공사 크로스체크: 청약홈 vs K-APT vs DB 불일치        │
│   ⑤ 중복 현장: 이름+지역 유사도 90% 이상 → 병합 후보       │
│   → 매주 월요일                                          │
└─────────────────────────────────────────────────────────┘
```

## 4. 데이터 검증 시스템 — 잘못된 데이터가 사용자에게 도달하지 않도록

### 4.1 3단계 검증 게이트

```
Gate 1: 수집 시점 검증 (Ingestion Validation)
  ┌──────────────────────────────────┐
  │ 크롤러가 데이터를 가져올 때       │
  │ → 타입 체크 (숫자인데 문자?)      │
  │ → 범위 체크 (가격 < 0?)          │
  │ → NULL 체크 (필수 필드 누락?)     │
  │ → 통과 못하면 DB에 안 넣음        │
  └──────────────────────────────────┘
           ↓
Gate 2: 저장 후 교차 검증 (Cross-Validation)
  ┌──────────────────────────────────┐
  │ DB에 저장된 후                    │
  │ → 다른 소스와 교차 비교           │
  │   (네이버 vs KRX vs Yahoo)       │
  │ → 전일 대비 급변 탐지 (±50%)     │
  │ → 이상치 → data_quality_alerts   │
  └──────────────────────────────────┘
           ↓
Gate 3: AI 생성 전 검증 (Pre-Analysis Validation)
  ┌──────────────────────────────────┐
  │ AI 분석 텍스트 생성 직전          │
  │ → 시총 0이면 생성 스킵            │
  │ → PER NULL이면 "PER 정보 없음"    │
  │   대신 "PER 데이터 수집 중"       │
  │ → 전일 대비 가격 변동 반영         │
  │ → data_quality_score 50 미만      │
  │   → 분석 생성 보류                │
  └──────────────────────────────────┘
```

### 4.2 데이터 품질 점수 (Data Quality Score)

각 종목/현장에 0~100점 품질 점수 부여:

```
주식 품질 점수 산출:
  가격 있음:        +20
  시총 정상 (>0):   +15
  PER/PBR 있음:     +15
  EPS/ROE 있음:     +15
  배당 정보 있음:    +10
  섹터 있음:        +10
  52주 고저 있음:    +10
  설명 있음:        +5
  ─────────────────
  만점:             100

부동산 품질 점수 산출:
  이름/지역:        +15
  시공사 있음:      +10
  세대수 있음:      +10
  분양가 있음:      +15
  입주예정 있음:    +10
  인근역 있음:      +15
  학군 있음:        +10
  좌표 있음:        +10
  설명 있음:        +5
  ─────────────────
  만점:             100

활용:
  score >= 70: AI 분석 생성 대상
  score 50-69: 분석 생성하되 "일부 정보 수집 중" 면책
  score < 50:  분석 생성 보류, 데이터 보강 우선
```
