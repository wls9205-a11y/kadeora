### 1.3 솔직한 피드백

**종목 페이지:** 가격 데이터는 정확하지만 투자 판단에 필요한 핵심 지표(PER/PBR)가 전무하고, AI 분석도 5%에 불과. 검색에 노출되더라도 사용자가 "데이터가 부실하네" 하고 이탈할 가능성 높음. 분석 텍스트에 잘못된 시총이 포함된 것도 신뢰도를 떨어뜨림.

**현장 페이지:** 구조화 데이터는 훌륭하게 갖춰져 있지만(7종 JSON-LD), 정작 AI 분석 텍스트가 26%밖에 없고 인근역(0.5%)/학군(2.1%)이 거의 비어있어서 "역세권 분석", "학군 비교" 같은 핵심 검색어에 대응 불가. 클러스터 블로그 0건은 가장 큰 문제 — 메인 페이지 1개만으로는 검색 결과 1페이지를 장악할 수 없음.

## 3. 실행 설계 — 전체 파이프라인

### Phase A: 데이터 정확성 확보 (우선순위 1 — 즉시)

이미 구축 완료된 크론들이 자동 실행됨:

| 크론 | 스케줄 | 대상 | 일일 처리량 | 전체 완료 예상 |
|------|--------|------|-----------|-------------|
| stock-fundamentals-kr | 평일 08:30 | KR PER/PBR/배당 | 50건 | ~26일 |
| stock-fundamentals-us | 매일 07:00 | US PER/PBR/시총 | 30건 | ~16일 |
| apt-enrich-location | 매일 10:00 | 인근역/학군/편의 | 30건 | ~190일 |
| data-quality-monitor | 매일 19:00 | 품질 점수 재산출 | 전체 | 매일 |

**개선 제안:** `apt-enrich-location` 처리량 30건/일 → 100건/일 (190일→57일)
카카오맵 API 일일 한도 10만건이므로 충분.

### Phase B: AI 분석 텍스트 대량 생성 (우선순위 1 — 즉시)

| 크론 | 스케줄 | 대상 | 일일 처리량 | 현재 | 목표 |
|------|--------|------|-----------|------|------|
| apt-analysis-gen | 매시간 :05 | 부동산 분석 | 360건/일 | 1,476건 | 5,728건 |
| stock-analysis-gen | 매시간 :35 | 주식 분석 | 360건/일 | 90건 | 1,805건 |
| batch-analysis-submit | 매일 04시 | Batch 500건 | 500건/일 | - | 전체 |
| batch-analysis-poll | 10분마다 | Batch 결과 | - | - | - |

**예상 비용:** Haiku 4.5 기준
- 부동산 4,252건 × 2000토큰 = ~$6
- 주식 1,715건 × 1500토큰 = ~$4
- **총 ~$10**으로 전체 완료 가능

### Phase C: 클러스터 블로그 대량 생성 (우선순위 1 — 즉시)

| 크론 | 스케줄 | 구조 | 일일 생산 | 목표 |
|------|--------|------|----------|------|
| blog-apt-cluster | 매일 07시 | 현장당 3편 (청약전략/분양가분석/입주비용) | 6편/일 | 인기 현장 500개 × 3편 = 1,500편 |
| blog-stock-cluster | 매일 08시 | 종목당 3편 (투자전략/배당분석/실적전망) | 6편/일 | 인기 종목 300개 × 3편 = 900편 |

**문제:** 일 6편 속도로는 500현장 × 3편 = 250일 소요.
**개선 제안:** 
- batch-cluster-submit 크론 신설 → Batch API로 일 50현장 × 3편 = 150편/일
- 500현장 × 3편 = 10일 완료
- 비용: 1,500편 × Haiku 2500토큰 = ~$5

### Phase D: 이미지 인포그래픽 자동 생성 (우선순위 2)

새로운 OG 이미지 엔드포인트 3종:

**D-1. 주가/분양가 차트 이미지**
```

/api/og-chart?apt=한화포레나-부산당리  → 분양가 범위 바 차트
```
- ImageResponse로 서버사이드 렌더링
- 가격 데이터 DB에서 실시간 반영
- alt: "삼성전자 2026년 주가 차트 현재가 193,100원"

**D-2. 핵심 지표 인포그래픽**


**이미지 사이트맵 확장:**
현재 OG 이미지 2장 → 종목/현장당 **5장**으로 확장

### Phase E: 비교 페이지 자동 생성 (우선순위 2)

**E-1. 주식 비교 페이지**
```
/stock/005930/vs/000660 → 삼성전자 vs SK하이닉스
```
- 같은 섹터 종목 자동 매칭
- 가격, PER, PBR, 배당, 시총 비교 테이블
- JSON-LD: ComparisonTable (비공식이지만 구글 이해 가능)
- "삼성전자 SK하이닉스 비교" 검색어 독점

**E-2. 부동산 비교 페이지**
```
/apt/한화포레나-부산당리/vs/래미안-부산당리
```
- 같은 지역 현장 자동 매칭
- 분양가, 세대수, 시공사, 입주일, 교통점수 비교
- "한화포레나 래미안 비교" 검색어 독점

**자동 매칭 로직:**
- 주식: 같은 섹터 + 시총 유사 (±50%) 상위 3개
- 부동산: 같은 시군구 + 6개월 이내 입주 상위 3개

### Phase F: 계산기 도구 전용 URL (우선순위 2)

현재 `/calc` 페이지는 존재하지만 종목/현장별 파라미터화가 안 됨.

**F-1. 주식 계산기**
```
/calc?stock=005930&type=return    → 삼성전자 투자 수익률 계산기
/calc?stock=005930&type=dividend  → 삼성전자 배당 재투자 시뮬레이터
/calc?stock=005930&type=dca       → 삼성전자 적립식 투자 계산기
```

**F-2. 부동산 계산기**
```
/calc?apt=한화포레나-부산당리&type=movein  → 실입주 비용 계산기
/calc?apt=한화포레나-부산당리&type=loan    → 대출 한도 계산기
/calc?apt=한화포레나-부산당리&type=tax     → 취득세 계산기
```

- 각 URL에 고유 title + meta description
- WebApplication JSON-LD 스키마
- 계산기는 체류시간이 3~5분으로 길어서 SEO 점수 ↑↑

### Phase G: 지역 허브 페이지 (우선순위 3)

```
/apt/region/부산-사하구   → 부산 사하구 전체 분양/시세 허브
/apt/region/경기-화성시   → 화성시 전체 분양/시세 허브
/stock/sector/반도체      → 반도체 섹터 전체 종목 허브
/stock/sector/자동차      → 자동차 섹터 전체 종목 허브
```

- "부산 사하구 분양" → 지역 허브가 개별 현장과 별도로 노출
- "반도체 관련주" → 섹터 허브가 개별 종목과 별도로 노출
- 내부 링크 허브 역할 → 모든 하위 페이지의 SEO 상승

### Phase H: 네이버 전용 최적화 (우선순위 3)

**H-1. RSS 피드 생성**
```
/feed.xml     → 전체 피드 RSS
/blog.xml     → 블로그 RSS (네이버 VIEW 탭 노출)

```
네이버 서치어드바이저에 RSS 등록 → VIEW 탭 진입

**H-2. 네이버 신디케이션 API**
- 새 블로그/분석 발행 시 자동으로 네이버에 통보
- IndexNow보다 네이버에 특화된 방식

**H-3. 페이지당 이미지 6장 이상**
- 네이버 블로그 상위 노출 기준: 이미지 6~13장
- 현장 페이지: 갤러리 + 인포그래픽 + 지도 + 차트 = 6장+
- 종목 페이지: 차트 + 인포그래픽 + 섹터맵 + 비교 = 6장+

## 6. 내부 링크 전략 — 모든 페이지가 서로 연결

```
종목 페이지 ──→ 클러스터 블로그 3편
     │              ↕
     ├──→ 비교 페이지 ──→ 경쟁 종목 페이지
     │              ↕
     ├──→ 계산기 도구 ──→ 종목 페이지 (역링크)
     │              ↕
     └──→ 섹터 허브 ──→ 같은 섹터 종목들

현장 페이지 ──→ 클러스터 블로그 3편
     │              ↕
     ├──→ 비교 페이지 ──→ 경쟁 현장 페이지
     │              ↕
     ├──→ 계산기 도구 ──→ 현장 페이지 (역링크)
     │              ↕
     └──→ 지역 허브 ──→ 같은 지역 현장들

피드 게시글 ──→ 종목/현장 자동 링크 (content-renderer)
     ↕
  댓글 ──→ 관련 종목/현장 자연 언급
```

**PageRank 흐름:** 피드(UGC) → 현장/종목 → 클러스터 블로그 → 허브
모든 페이지가 순환 연결 → 사이트 전체 권위 상승

## 7. AI 분석 텍스트 품질 개선

### 현재 문제

AI 분석이 서술형으로 작성되어 있어서:
1. AI Overview에 인용되기 어려움 (구조화 안 됨)
2. Featured Snippet 진입 불가 (핵심 답변 없음)
3. PAA 답변으로 부적합

### 개선: 데이터 테이블 + 핵심 수치 선행 구조

```markdown
## 핵심 투자 지표

| 항목 | 수치 | 판정 |
|------|------|------|
| 현재가 | 193,100원 | +3.71% |
| PER | 9.2배 | 업종 평균 15.3배 대비 저평가 |
| PBR | 1.8배 | 적정 수준 |
| 배당수익률 | 1.8% | 업종 상위 |
| 시가총액 | 1,044조원 | 코스피 1위 |
| 52주 범위 | 52,900 ~ 223,000원 | 현재 상단 |

## Q. 삼성전자 PER은 얼마인가요?
삼성전자의 PER은 9.2배로, 반도체 업종 평균 15.3배 대비 저평가 구간에 있습니다.

## Q. 삼성전자는 배당금을 주나요?
삼성전자는 연 배당수익률 1.8%로, 분기 배당을 실시합니다.
```

이 구조로 변경하면:
- 구글 AI Overview가 테이블 데이터 직접 인용
- Featured Snippet에 표 형태로 노출
- PAA 답변으로 각 Q&A 섹션 활용

### 프롬프트 개선

```
기존: "종합 분석 2000자+"
개선: "핵심 투자 지표 테이블(현재가/PER/PBR/배당/시총/52주) 먼저,
      그 다음 Q&A 형식 5개(각 2~3문장),
      마지막 종합 의견 500자"
```
