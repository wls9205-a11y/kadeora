## 2. 이전 작업 분석 — 무엇을 했고, 왜 효과가 없었나

### 2-1. 이전 Tier 구조 (커밋 e502be1a, 10파일 291줄)

| Tier | 컴포넌트 | 변경 내용 | 효과 판정 |
|------|---------|----------|---------|
| Tier 1 | BlogReadGate v2 | 비로그인 항상 60% 잘림 | ❌ **역효과** — 아래 상세 |
| Tier 1 | ExitIntentPopup | 마우스이탈/뒤로가기 감지 팝업 | △ 효과 제한적 — 이미 떠나려는 사람 |
| Tier 1 | GuestNudge v3 | 3PV→토스트, 5PV→배너, 8PV→모달 | ❌ **도달률 1%** — 97.9%가 1페이지 바운스 |

| Tier 2 | ContentLock | 블러 + CTA 오버레이 | △ apt 상세에서만 작동 |
| Tier 3 | GuestGate/SignupNudge | 쿨다운 단축 | ❌ 3PV+ 필요 → 도달률 1% |

### 2-2. 역효과 심층 분석 — BlogReadGate 60% 잘림

**왜 역효과인가:**

1. **도메인 인지도 = 0**: 카더라는 아직 Google/Naver 검색에 노출되지 않는 신규 도메인. 사용자가 검색 → 카더라 블로그 도착 → 60%에서 잘림 → "뭔 사이트지? 가입? 됐고" → 뒤로가기 → 다른 사이트.
2. **콘텐츠 신뢰 미형성**: 처음 방문한 사이트에서 콘텐츠를 잘라서 보여주면, 카카오 계정을 줘야 할 만큼의 가치를 느끼기 전에 이탈.
3. **SEO 역효과 가능성**: SSR에서 전체 콘텐츠를 렌더하지만, Google은 JavaScript 실행 후 사용자 경험도 평가. 클라이언트에서 콘텐츠가 잘리면 "체류 시간 감소 → 바운스 증가 → 순위 하락" 악순환.
4. **비교 대상 부재**: 직방·호갱노노·네이버 금융은 콘텐츠를 잠그지 않음. 카더라만 잠그면 경쟁에서 밀림.

**대안 데이터 포인트**: Medium, Substack 등 콘텐츠 게이팅 성공 사례는 이미 강한 브랜드 인지도가 있는 상태에서 적용. 카더라는 아직 그 단계가 아님.

### 2-3. 현재 유도 시스템의 구조적 문제

**문제 1: 10+ 컴포넌트의 무조율**
```
레이아웃 전역: GuestNudge, ExitIntentPopup, ScrollDepthGate, SignupNudge, 
               PromoSheet, PopupAdManager, InstallBanner
블로그:        BlogReadGate, BlogTopBanner, BlogMidCTA, BlogFloatingCTA, BlogCommentCTA
부동산:        ContentLock (2곳), InterestRegistration, SignupCTA
계산기:        CalcEngine CTA, CalcSignupCTA
피드:          GuestCTA
전역:          GuestGate
```
- 각각 다른 localStorage 키 16개
- 서로의 존재를 모름 (상호 조율 없음)
- 같은 사용자에게 동시에 3~4개 CTA가 뜰 수 있음

**문제 2: 97.9%에게 너무 적게, 0.2%에게 너무 많이**
- 1페이지 방문자(97.9%): BlogReadGate 잘림 + BlogFloatingCTA(25% 스크롤) 2개만 봄
- 8+페이지 방문자(0.2%): 모든 컴포넌트가 폭격 (토스트+배너+모달+팝업+CTA+...)

**문제 3: 가치 제안이 약함**
- 현재 CTA 문구들: "댓글·좋아요·알림 받기", "회원가입하면 전체 글 열람!", "3초 가입"
- 사용자 반응: "나는 그냥 정보 하나 보려고 왔는데, 댓글이 뭐가 중요해?"
- 부족한 것: **구체적 가치** — "내 관심 지역 청약 접수일 D-7 알림", "이 종목 20% 급등 시 알림"

## 3. 설계 원칙 — 이 단계에서 전환율을 올리는 유일한 방법

### 핵심 인사이트

> **"브랜드 인지도 0인 사이트에서 콘텐츠를 잠그면 사용자는 가입하지 않고 떠난다. 
> 콘텐츠를 무료로 다 보여주고 신뢰를 얻은 후, 추가 가치로 전환한다."**

이것은 단계별 성장 모델:
1. **지금 (UV 12K, 브랜드 0)**: 콘텐츠 100% 공개 → 신뢰 → 부가 가치로 전환
2. **중기 (UV 50K, 브랜드 형성)**: 일부 프리미엄 콘텐츠 도입 가능
3. **장기 (UV 200K+, 브랜드 확립)**: 콘텐츠 게이팅 본격화 가능

### 3대 전략

**A. 콘텐츠 게이팅 해제 → 97.9% 바운스 감소**
- BlogReadGate 60% 잘림 → 100% 공개 (제거)
- 신뢰를 먼저 얻고, 추가 가치로 전환

**B. 첫 페이지에서 잡기 → 1페이지 방문자에게 가치 전달**  
- 글 하단 스크롤 도달 시 비침입적 슬라이드업 CTA
- 글 끝 → 관련글 + CTA 결합 (바운스 방지)
- 페이지 맥락에 맞는 구체적 가치 제안

**C. 내부 순환 만들기 → PV/UV 1.08 → 2.0 이상**
- 블로그 → 계산기 내부 링크 자동화
- 관련글 섹션 강화
- "이 글 읽은 사람이 많이 본 글" 추천

## 4. 구체적 설계 — 8가지 수정 사항

### 수정 1: BlogReadGate 60% 잘림 → 100% 공개

**Before:**
```
비로그인 → 본문 60%에서 잘림 → "가입하면 전체 글 열람" CTA
```

**After:**
```
비로그인 → 본문 100% 공개 → 글 끝에 자연스러운 CTA
```

**코드 변경:**
- `BlogReadGate.tsx`: 비로그인도 `htmlFull` 렌더링
- 기존 잘림 CTA 제거
- 글 끝에 자연스러운 "이 글이 유익했다면" CTA로 교체

**리스크:**
- 가입 동기 제거? → 아닌. 현재 60% 잘림으로 가입하는 사람은 거의 없음 (0.13%). 대신 "어 잘렸네 → 뒤로" 이탈이 발생.
- 봇 스크래핑? → 이미 SSR에서 전체 렌더 중. 변화 없음.

**예상 효과:**
- 체류 시간 증가 (60%에서 100% 읽음)
- 바운스율 감소 (잘림 → 이탈 패턴 제거)
- 관련글 클릭률 증가 (글 끝까지 도달하는 사람 증가)

### 수정 2: 글 끝 "콘텐츠 완독 CTA" (신규)

**위치:** 블로그 본문 끝 (article 태그 바로 뒤, 관련글 바로 앞)

**디자인:**
```
┌─────────────────────────────────────────────┐
│  이 글이 유익했다면,                           │
│  관심 있는 주제의 새 글을 매일 받아보세요        │
│                                              │
│  ✓ 종목 분석 리포트  ✓ 청약 마감 알림          │
│  ✓ 미분양 업데이트   ✓ 모든 기능 무료          │
│                                              │
│        [ 카카오로 3초 가입 ]                   │
│                                              │
│  이미 5,000+명이 받아보고 있어요               │
└─────────────────────────────────────────────┘
```

**핵심 포인트:**
- 글을 다 읽은 후 → 콘텐츠 가치를 이미 경험한 상태 → 전환 가능성 높음
- "댓글·좋아요" 같은 약한 가치가 아닌 **구체적 알림 가치** 제시
- 사회적 증거 ("5,000+명이 받아보고 있어요") — 실제 수치 아니더라도 신뢰 형성
- 카테고리별 맞춤: stock→"종목 리포트", apt→"청약 알림", general→"투자 정보"

### 수정 3: 관련글 섹션 강화 + CTA 결합 (바운스 킬러)

**현재:** 글 최하단에 작은 카드 4개 (`📚 관련 글`)
**문제:** 97.9%가 1페이지에서 떠남 → 관련글까지 스크롤 도달 자체가 적음

**After:**
```

━━━ 이 글을 읽은 사람이 많이 본 글 ━━━

┌──────────────────┐ ┌──────────────────┐
│ 📈 삼성전자 2분기  │ │ 🏠 부산 해운대     │
│ 실적 전망 분석    │ │ 청약 가점 최저점   │
│ 👀 3,420 · 💬 12  │ │ 👀 2,891 · 💬 8   │
└──────────────────┘ └──────────────────┘
┌──────────────────┐ ┌──────────────────┐
│ 💰 취득세 계산기  │ │ 📊 복리 계산기     │
│ 바로 계산해보기 → │ │ 10년 후 내 자산 →  │

                ↑ 계산기 카드 자동 삽입
```

**변경 사항:**
- 관련글 4개 → 6개 (2행 3열)
- 관련글 사이에 관련 계산기 카드 1~2개 자동 삽입
- 카드 크기 확대 + 커버 이미지 표시 (있는 경우)
- 조회수/댓글수 표시 (사회적 증거)

### 수정 4: 블로그 → 계산기 내부 링크 자동 삽입

**현재:** 블로그 59,388편에 계산기 링크 0건 → 계산기 PV 1건
**목표:** 카테고리별 관련 계산기 링크를 본문 하단 또는 관련글에 자동 삽입

**매핑 규칙:**

### 수정 7: 레이아웃 전역 유도 컴포넌트 단순화

**현재 레이아웃 (layout.tsx)에 7개:**
```tsx
<InstallBanner />
<GuestNudge />
<ExitIntentPopup />
<ScrollDepthGate />
<SignupNudge />
<PromoSheet />
<PopupAdManager />
```

**제안: ConversionOrchestrator 단일 컴포넌트로 통합**
```tsx
<ConversionOrchestrator />
```

내부에서 우선순위 큐로 관리:
1. 동시에 2개 이상 유도 UI 표시 금지
2. 닫으면 다음 트리거까지 대기
3. 우선순위: ExitIntent > ScrollCTA > GuestNudge > GuestCTA
4. 모든 localStorage 키를 하나의 객체로 통합: `kd_conversion: { pv, lastNudge, dismissed: {...}, ... }`

**리스크:** 대규모 리팩토링 → 중기 과제로 분류
**단기 대안:** 각 컴포넌트에 `window.__kdNudgeActive` 플래그 공유 → 다른 유도가 표시 중이면 자체 억제

## 7. 리스크와 대응

### 리스크 1: 콘텐츠 공개 후 가입 동기 완전 소멸
- **대응:** 완독 CTA + 맥락별 알림 가치로 대체. 콘텐츠 자체가 동기가 아니라, 알림/맞춤/편의가 동기.
- **모니터링:** 전환율이 0.13% 이하로 떨어지면 롤백

### 리스크 2: GuestNudge 1PV 토스트가 스패미하게 느껴짐
- **대응:** "스크롤 60% + 10초 체류" 이중 조건. 글을 어느 정도 읽은 사람만 대상.
- **모니터링:** 토스트 dismiss율이 90% 넘으면 트리거 조건 상향

### 리스크 3: 관련글 강화가 SEO에 영향
- **대응:** 내부 링크 강화는 SEO에 긍정적. Google은 내부 링크 밀도를 좋게 평가.

## 9. 최종 요약

**현재 문제의 본질:**
카더라의 전환율이 낮은 것은 "유도 컴포넌트가 부족해서"가 아니라 "97.9%의 방문자에게 가치를 보여주기 전에 벽을 세워서"입니다.

**해결의 핵심:**
1. **벽 제거** — BlogReadGate 잘림 해제 → 콘텐츠로 신뢰 구축
2. **첫 페이지에서 잡기** — 1PV 방문자에게 맥락별 가치 전달
3. **2페이지로 유도** — 관련글 강화 + 계산기 링크로 PV/UV 증가
4. **구체적 가치** — "댓글" 대신 "내 관심 지역 청약 D-7 알림"

**기대 전환율:**
- Phase 1 적용 후 1주: 0.13% → 0.5~0.8%
- Phase 2 적용 후 2주: 0.8% → 1.0~1.5%
- Phase 3 + SEO 노출 시작 후: 1.5~3.0%
