# 카더라 CTA 시스템 완전 재설계안

> 기존 12개 CTA 컴포넌트 전면 백지화 → 단일 통합 시스템으로 재설계
> 작성: 세션 77 | 2026-04-07

---

## 1. 기존 시스템 부검 — 왜 실패했는가

### 1.1 핵심 데이터

```
방문자 행동 (7일, 8,489 UV):
  97.5% → 1페이지 바운스 (8,279명)
  98.0% → 체류 10초 미만
   1.3% → 2페이지 (107명)
   1.2% → 3페이지+ (103명)

CTA 성과:
  노출률: 0.78% (66/8,490)
  클릭률: 0% (유일한 클릭 = "닫기")
  기여 가입: 0명 (14명 가입은 전부 CTA 무관)
```

### 1.2 실패 근본 원인

**기존 CTA 시스템은 "다중 페이지 세션"을 전제로 설계됨.**
그러나 현실은 97.5%가 1페이지에서 이탈.

| 설계 전제 | 현실 | 괴리 |
|----------|------|------|
| 2PV 이상에서 CTA 노출 | 97.5%가 1PV | CTA를 볼 기회 자체가 없음 |
| 재방문자에게 에스컬레이션 | 재방문율 극히 낮음 | 에스컬레이션 대상 없음 |
| 스크롤 50%에서 플로팅 CTA | 대부분 스크롤 안 함 | 플로팅 CTA 미노출 |
| 10초 체류 후 Exit Intent | 체류 10초 미만 98% | Exit Intent 트리거 불가 |

**기존 시스템의 12개 CTA 컴포넌트:**
GuestNudge, SmartSectionGate, TwoStepCTA, BlogTopBanner, 
BlogMidCTA, BlogFloatingCTA, ReturnVisitorBanner, ExitIntentPopup,
BlogReadGate, AutoPushPrompt, NewsletterSubscribe, ScrollDepthGate

→ 12개가 각자 localStorage 관리, 복잡한 오케스트레이터로 조율
→ 결과: 복잡도만 높고 성과 0

### 1.3 자연 가입 경로 분석 (30일 20명)

```
CTA 없이 자발적으로 가입한 사용자의 마지막 페이지:
  /blog/*     7명 (35%) — 블로그 콘텐츠에 관심
  /apt/*      6명 (30%) — 부동산 현장 정보에 관심  
  /feed/*     5명 (25%) — 커뮤니티 토론에 참여 욕구
  /stock/*    2명 (10%) — 종목 정보에 관심
```

**핵심 인사이트:** 사용자는 CTA 때문이 아니라 "콘텐츠가 유용해서" 가입함.
CTA의 역할은 "가입 유도"가 아니라 **"가입 경로를 쉽게 만들어주는 것"**.

## 2. 새로운 설계 철학

### 2.1 5가지 원칙

```
1. ONE PAGE RULE
   → 97.5%가 1페이지만 봄. 모든 CTA는 첫 페이지 내에서 작동해야 함.
   → 2PV 이상 조건 완전 폐기.

2. CONTENT-NATIVE
   → CTA는 콘텐츠의 일부처럼 보여야 함.
   → 팝업/모달/오버레이 최소화. 인라인 카드가 기본.
   → 사용자는 "이건 광고"라고 느끼면 무시함.

3. CONTEXT-SPECIFIC
   → "주식 알림 받기" (X) → "삼성전자 급등 알림 받기" (O)
   → 현재 보고 있는 종목명/현장명/블로그 주제가 CTA에 반영.

4. VALUE-BEFORE-ASK
   → 먼저 유용한 정보를 보여주고, 그 연장선에서 가입 유도.
   → "더 자세한 분석은 가입 후 확인" (X)
   → "이 종목 실시간 알림도 무료로 받아보세요" (O)

5. SINGLE SYSTEM
   → 12개 컴포넌트 → 1개 통합 컴포넌트.
   → 오케스트레이터 불필요. 컴포넌트 자체가 맥락을 판단.
```

### 2.2 새 아키텍처 개요

```
기존: 12개 컴포넌트 + ConversionOrchestrator + 복잡한 localStorage
신규: 2개 컴포넌트 (InlineCTA + StickyBar)

┌─────────────────────────────────────────┐
│              페이지 콘텐츠               │
│                                         │
│  ┌─────────────────────────────────┐    │
│  │  📊 InlineCTA (콘텐츠 내장형)    │    │  ← 스크롤 없이 보임
│  │  "삼성전자 가격 변동 알림 받기"    │    │
│  │  [카카오로 무료 시작]             │    │
│  └─────────────────────────────────┘    │
│                                         │
│  ... 나머지 콘텐츠 ...                   │
│                                         │
│  ┌─────────────────────────────────┐    │
│  │  🔒 ValueLock (핵심 섹션 블러)    │    │  ← 스크롤 시 보임
│  │  "AI 분석 전문은 가입 후 확인"    │    │
│  └─────────────────────────────────┘    │
│                                         │
└─────────────────────────────────────────┘
┌─────────────────────────────────────────┐
│  StickyBar (하단 고정)                   │  ← 8초 후 슬라이드 업
│  "삼성전자 알림 받기" [무료 가입]         │
└─────────────────────────────────────────┘
```

## 3. 새 컴포넌트 상세 설계

### 3.1 InlineCTA — 콘텐츠 내장형 전환 카드

**목적:** 콘텐츠 흐름 안에 자연스럽게 삽입되어, 읽는 도중 가입 동기 생성

**배치 위치:**
- 블로그: 본문의 40~60% 지점 (첫 번째 주요 섹션 끝)
- 부동산: 분양가 정보 바로 아래
- 주식: KPI 그리드 바로 아래
- 피드: 3번째 게시글과 4번째 게시글 사이

**디자인 원칙:**
- 카드 형태 (팝업 아님)
- 주변 콘텐츠와 동일한 배경색
- 닫기 버튼 없음 (콘텐츠의 일부이므로)
- 아이콘 + 한 줄 가치 제안 + 버튼

**맥락별 메시지:**

```
블로그 (주식 카테고리):
  📈 이 종목 시세가 변동하면 알림으로 알려드립니다
  [무료 알림 받기]

블로그 (부동산 카테고리):  
  🏠 이 단지 청약 마감 D-7 알림을 받아보세요
  [무료 알림 받기]

주식 상세 (/stock/005930):
  📊 삼성전자 ±3% 변동 시 즉시 알림
  현재가 193,100원 기준 | 카카오 3초 가입
  [알림 설정하기]

부동산 상세 (/apt/힐스테이트):
  🔔 힐스테이트 부산 청약 알림 받기
  접수 시작 D-7 · 당첨 발표 · 분양가 변동
  [무료 알림 받기]

피드:
  💬 이 토론에 의견을 남겨보세요
  [카카오로 3초 참여]
```

**핵심 차별점 (기존 BlogMidCTA 대비):**
- 기존: 항상 같은 메시지, 본문 중간에 끼워넣기
- 신규: 페이지 데이터(종목명, 현장명, 가격)를 실시간 반영
- 기존: 팝업처럼 보이는 분리된 카드
- 신규: 주변 콘텐츠와 시각적으로 통합된 자연스러운 카드

**구현:**
```tsx
// 서버 컴포넌트에서 데이터를 props로 전달
<InlineCTA 
  type="stock"
  entityName="삼성전자"
  entityId="005930"
  currentPrice={193100}
  changePercent={-1.2}
/>
```

### 3.2 StickyBar — 하단 고정 바

**목적:** 스크롤 위치와 무관하게 항상 접근 가능한 전환 경로

**트리거 조건:**
```
비로그인 + 페이지 로드 후 8초 경과 → 슬라이드 업 애니메이션
```

- 2PV 조건 없음 (1페이지에서 작동)
- 세션당 1회 닫으면 해당 세션 동안 미표시
- 다음 방문 시 다시 표시

**디자인:**
```
┌──────────────────────────────────────────────┐
│  📊 삼성전자 알림 받기   [카카오 무료 가입]  ✕  │  높이 48px
└──────────────────────────────────────────────┘
```

- 높이 48px (기존 GuestNudge 배너보다 작음)
- 네비게이션 바 바로 위에 위치
- 투명도 95% 배경 (콘텐츠 약간 비침)
- 맥락별 메시지 (InlineCTA와 동일한 엔티티 데이터 사용)

**기존 GuestNudge 대비:**
- 기존: 2PV+ 조건, 토스트→배너→모달 에스컬레이션
- 신규: 8초 후 무조건 표시, 에스컬레이션 없음 (바만 있음)
- 기존: "주식 알림 · 청약 알림 · 실시간 토론" (일반적)
- 신규: "삼성전자 알림 받기" (맥락 맞춤)

### 3.3 ValueLock — 핵심 섹션 블러 (SmartSectionGate 개선)

**기존 SmartSectionGate 유지하되 개선:**

1. 헤딩 패턴 확장 (12개 → 25개+)
2. 블러 영역에 "미리보기" 효과 강화
3. CTA 메시지 맥락 맞춤화

```
추가 헤딩 패턴:
  '핵심 요약', '결론', '투자 전략', '매매 전략', '체크리스트',
  '비교 분석', '수익률', '실전 적용', '입주 전략', '절세 전략',
  '리스크 분석', '매수 타이밍', '적정 가격', '수익 시뮬레이션',
  '최종 판단'
```

## 4. 제거 대상 (백지화)

### 4.1 완전 제거 (7개)

| 컴포넌트 | 제거 이유 |
|---------|----------|
| **GuestNudge** | 2PV+ 조건으로 99%에게 미노출. StickyBar로 대체 |
| **ExitIntentPopup** | 10초 체류 필요하나 98%가 10초 미만. 0회 트리거 |
| **ReturnVisitorBanner** | 재방문자 극소수. 6회 노출, 0 클릭 |
| **TwoStepCTA** | 2단계는 10초 방문에 과다. InlineCTA로 대체 |
| **BlogTopBanner** | InlineCTA가 더 자연스럽게 대체 |
| **BlogFloatingCTA** | StickyBar가 대체 |
| **ConversionOrchestrator** | 2개 컴포넌트로 단순화되어 조율 불필요 |

### 4.2 유지 + 개선 (2개)

| 컴포넌트 | 변경 |
|---------|------|
| **SmartSectionGate** → **ValueLock** | 헤딩 패턴 확장 + 메시지 맥락화 |
| **BlogMidCTA** → **InlineCTA** | 완전 재작성. 맥락 데이터 주입 |

### 4.3 유지 (변경 없음)

| 컴포넌트 | 이유 |
|---------|------|
| **AutoPushPrompt** | 로그인 유저 대상 (전환과 무관) |
| **BlogReadGate** | SmartSectionGate 래퍼 (유지) |

## 5. 데이터 흐름 설계

### 5.1 맥락 데이터 전달

```
서버 컴포넌트 (SSR)
  ↓ 종목명/현장명/카테고리/가격 추출
  ↓
클라이언트 컴포넌트 (InlineCTA, StickyBar)
  ↓ props로 맥락 데이터 수신
  ↓
CTA 메시지 동적 생성
  ↓
trackConversion 이벤트 기록
```

### 5.2 이벤트 추적 (단순화)

```
기존: 4단계 (cta_view → cta_click → cta_step2 → cta_complete)
신규: 3단계 (impression → click → signup)

conversion_events 테이블:
  event_type: 'impression' | 'click' | 'signup'
  cta_name: 'inline_cta' | 'sticky_bar' | 'value_lock'
  entity_type: 'stock' | 'apt' | 'blog' | 'feed'
  entity_name: '삼성전자' | '힐스테이트 부산' | null
  page_path: '/stock/005930'
  visitor_id: 'uuid'
```

### 5.3 어드민 대시보드 표시

```
기존: "관심등록 1, 전환 3" (의미 불명확)
신규: 
  ┌──────────────────────────────────┐
  │  CTA 퍼널 (7일)                  │
  │  노출 1,200 → 클릭 48 → 가입 12 │
  │  노출률 14% | CTR 4% | CVR 0.14% │
  └──────────────────────────────────┘
```

## 7. 예상 성과

### 7.1 보수적 추정

```
현재 → 개선 후:

노출률:  0.78%  →  35~50%
  InlineCTA: 모든 상세 페이지에 내장 (스크롤 없이 보임)
  StickyBar: 8초 체류 방문자 전원

클릭률:  0%  →  2~4%
  맥락 맞춤 메시지 ("삼성전자 알림" vs "주식 알림")
  콘텐츠 흐름에 통합 (광고로 인식 안 됨)

일일 가입:  2명  →  4~8명
  1,200 UV × 40% 노출 × 3% 클릭 × 50% 가입 완료 = ~7명

월간 가입:  60명  →  120~240명
```

### 7.2 핵심 KPI 목표

| 지표 | 현재 | 1주 후 | 1개월 후 |
|------|------|--------|---------|
| CTA 노출률 | 0.78% | 30%+ | 40%+ |
| CTA CTR | 0% | 2% | 3~5% |
| 일일 가입 | 2명 | 4명 | 6~8명 |
| CTA 기여 가입 비율 | 0% | 30%+ | 50%+ |

## 8. 구현 계획

### Phase 1: 기반 정리 (1시간)

```
1. 기존 CTA 컴포넌트 비활성화
   - GuestNudge: layout.tsx에서 렌더링 제거
   - ExitIntentPopup: layout.tsx에서 렌더링 제거
   - ReturnVisitorBanner: layout.tsx에서 렌더링 제거
   - TwoStepCTA: blog/apt에서 렌더링 제거
   - BlogTopBanner: blog에서 렌더링 제거
   - BlogFloatingCTA: blog에서 렌더링 제거
   - ConversionOrchestrator: Provider 제거
   
   ※ 파일 자체는 삭제하지 않음 (롤백 안전장치)
   ※ import + JSX 렌더링만 제거
```

### Phase 2: InlineCTA 구현 (1.5시간)

## 9. 리스크 관리

| 리스크 | 확률 | 대응 |
|--------|------|------|
| InlineCTA가 콘텐츠 가독성 저해 | 🟡 | 주변 콘텐츠와 동일 스타일, 경계선 최소화 |
| StickyBar가 모바일에서 탭바 가림 | 🟡 | 탭바 위 48px에 배치, safe-area-inset 적용 |
| 구글 인터스티셜 패널티 | 🟢 | 인라인 카드 + 슬림 바 = 비차단형 (패널티 대상 아님) |
| 기존 CTA 제거 후 가입 감소 | 🟢 | 현재 CTA 기여 가입 0명이므로 감소 리스크 없음 |
| 롤백 필요 시 | 🟢 | 기존 파일 삭제하지 않고 렌더링만 제거하므로 즉시 복원 가능 |

  ✅ CTA 노출률 > 30%
  ✅ CTR > 3%
  ✅ 월간 가입 > 100명
  ✅ CTA 기여 가입 비율 > 40%

실패 시:
  → InlineCTA 메시지 A/B 테스트
